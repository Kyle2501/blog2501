Dissecting Octopress; Creating a New Theme [Part 01]

Feb 18th, 2012

This series of posts will be written and updated in an ongoing process until we understand best how to create a new theme for Octopress. My girl friend is doing the design – I am doing the development. We both installed Octopress this week and are running it on GitHub Pages. Her’s can be found here uwenna.

For now these posts act as a brainstorm with myself… then will be broken up into better sections for publishing.

What is understood about Octopress themes so far:

There is a file called _config.yml which acts as a central reference place. This file controls data through out the site such as author name, titles, URL structure, plugin settings.

The theme is broken down into sections and put into individual .html and .scss files. These html files link and reference each other, using the { % -__- % format.

The theme files are stored in a hidden file called ‘.themes’ but are duplicated and can be accessed in the root folder.

A theme’s folder has two folders: ‘sass’ and ‘source’. ‘sass’ is the styling – ‘source’ is the structure.

How the default theme is built:
